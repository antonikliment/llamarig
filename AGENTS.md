# Agent Rules

## Go Receiver Style

- For any Go type, use either pointer receivers or value receivers for all methods on that type. Never mix both receiver styles on the same type.
- Prefer pointer receivers when methods mutate the receiver, the type contains synchronization primitives, or the type is large enough that copying is undesirable.
- Prefer value receivers only for small immutable value types.

## Quality Guard

- Implementation Go source must stay under 10,000 gocloc-counted code lines.
- Complexity, duplication, function size, and implementation LOC must pass `./custom-golangci-lint run`.
- Run `make test` and `make lint` so protobuf generation happens before checks.
- Never commit generated protobuf output under `core/rpc/gen/`; it is generated by `go generate ./core/rpc/gen`.
- Before adding logic, inspect existing packages and reuse the existing owner.

## Reuse-First Checklist

Before editing, answer:

1. What existing package owns this behavior?
2. Which existing types/functions can be reused or extended?
3. Why is new logic necessary?
4. What code can be removed or consolidated to stay under the LOC budget?
